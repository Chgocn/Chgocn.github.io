Fragment中请求网络报出NullPointException

  • OkHttpClient
  • fragment
Android

前言

今天同事在fragment中使用我封装的一个OkHttp请求网络时，死活报空指针，所以无奈他找到了我，然后向我求救。其中引发了我对Android的底层实现的进一步学习。

Bug还原

同事代码（这里只是拿了关键性代码）

public XxxFragment extends Fragment{
    private OkHttpCallBack mOkHttpCallBack = new OkHttpCallBack(getActivity){
        @Override
        public void onFailure(final Request request, final IOException e) {
            Log.e("TAG","on failure...");
        }

        @Override
        public void onResponse(final Response response) {
            Log.e("TAG","on success...")
        }
    }

    @Override
    public View onCreatView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState)) {
        View view = inflater.inflater(R.layout.xxx,container,false);
        RequestBody formBody = new FormEncodingBuilder()
        .add("phone", phone)
        .build();
        Request request = new Request.Builder()
        .url(getUrl("common/send-code"))
        .tag("sendCode")
        .post(formBody)
        .build();
        new OkHttpClient().newCall(request);
        return view;
    }
}

我封装的代码

public abstract class OkHttpCallback implements com.squareup.okhttp.Callback {

    private Activity mActivity;

    public OkHttpCallback(Activity activity) {
        this.mActivity = activity;
    }

    @Override
    public void onFailure(final Request request, final IOException e) {
        **mActivity.runOnUiThread(new Runnable() {
            @Override
            public void run() {
                onFailure(request, e);
            }
        });
    }

    @Override
    public void onResponse(final Response response) {
        mActivity.runOnUiThread(new Runnable() {
            @Override
            public void run() {
                onResponse(request, e);
            }
        });
    }

最后编译器直接定位到我用“**”标记的位置。然后同事傻傻的看着我，貌似在说：“丫的，你的代码有问题，去死吧！”，然后他又向我演示，他的网络请求在Activity里也是work的，就是在fragment里不行。好了，既然问题涉及到咱，咱也不能袖手旁观吧，好的，我们一起来分析。

分析

根据空指针原则，首先这个问题给我的第一感觉是activity传过来了么，虽然我在心里默默念了句，“丫的，你的代码肯定有问题！”，但是我还是很“耐心的”给他分析，你这activity应该是空的。于是乎，我想起来了fragment的生命周期，难道onCreateView()还没获取到activity对象？于是乎我让他把请求网络方法放到了onViewCreated()里。但是依然报错，于是乎打印log,debug,最终证明我的感觉是对的。他的getActivity()获取的activity为空。于是我让他把mOkHttpCallBack放入到onCeateView()中，而不是在成员变量中直接new 出来，于是work了。

深入研究

首先来看一下Fragment的生命周期。

查阅资料可知，Fragment的getActivity()只是在onAttach()和onDetach()之内才work,否则就会报NullPointException。
接下来我们来比较一下JAVA类和对象的初始化顺序。
类：类描述了具有相同特性（数据元素）和行为（功能）的对象集合，也是一个数据类型。比如说动物，它有重量、大小的特征（数据元素），也有移动、吃的行为（功能）。

对象：对象是具有状态、行为和标识的是上面类的一个实例。每一个对象都有内部数据(它描述该对象的状态)和方法（它产生行为），并且每一个对象都可以与其他对象区分开来，就是一个对象在内存中都有一个唯一的地址。在Java中对象都必须通过new来产生。

static：static表示“全局”或者“静态”的意思，用来修饰成员变量和成员方法，也可以形成静态static代码块。它是属于类不属于对象。

初始化函数：也就是构造器，构造器名称必须与类名完全相同，没有返回值。没有参数的构造器叫做默认构造器，你在不重载情况下都是调用的默认构造器。

类的加载顺序：static块和对象：无论写在类的前面还是后面都会先执行，多个静态块或者对象执行顺序与写的顺序一致，写在前面的先执行；非静态块和对象：按照在类中写的先后顺序执行；初始化函数；

最终分析

既然了解了Fragment的生命周期和JAVA类和对象的初始化顺序，那这个问题就很好解释了，当我们把请求网络的回调写在了成员变量位置，便直接new 出来，此时根本获取不到Activity对象，所以报NullPointException。
