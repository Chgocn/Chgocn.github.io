Issue:Android横竖屏切换导致activity重启

Android
  • issue

我的需求是禁止activity横屏。所以我在AndroidMenifest中将activity标签加入
android:screenOrientation=”portrait”，以此禁止横竖屏切换。

测试步骤：（TestApp只重写了生命周期方法）

1、进入“TestApp”首页（竖屏）；
2、进入后台，打开“优酷”，点击任意视频，重力感应全屏（目的是让手机横屏）；
3、点击“任务栏”，切换到“测试app”。

华为手机、三星手机现象：

1、onCreate() -> onStart() -> onResume()
2、onPause() -> onStop()
3、onRestart() -> onStart() -> onResume()

小米手机现象：

1、onCreate() -> onStart() -> onResume()
2、onPause() -> onStop()
3、onDestroy() -> 1 -> 2 -> onDestroy() -> 1

思考：

虽然“TestApp”中设置了默认portrait方式，但是进入“优酷”中，手机界面被“强制”横屏了，那么再切换到“TestApp”，activity的config信息已经被改变了？后来我重写了Application的生命周期方法，证实了这一点。

带着这样的思考，我又给activity标签加入了android:configChanges=”orientation”，MainActivity重写onConfigurationChanged(Configuration newConfig)方法。继续按照测试步骤执行。现象依然同上。

解决方案

设置orientaion的同时，如果API级别13或更高级别，还要设置sceenSize
android:configChanges=”orientation|screenSize”
